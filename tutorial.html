进阶 02
接入 Poki SDK：让游戏支持广告
▼
🎯 Poki 审核必须接入 SDK，4 个 API 就够了
第一步：引入 SDK（加在 <head> 里）
<script src="https://game-cdn.poki.com/scripts/v2/poki-sdk.js"></script>
第二步：4 个核心 API
PokiSDK.gameLoadingFinished()
🎮 游戏资源加载完毕时调用。告诉 Poki "游戏准备好了"
📍 触发时机：所有图片/音频加载完毕后
PokiSDK.gameplayStart()
▶️ 玩家开始实际游玩时调用（不是加载，是真正在玩）
📍 触发时机：点击"开始游戏"按钮后
PokiSDK.gameplayStop()
⏸️ 游玩暂停或结束时调用（Game Over 或打开菜单）
📍 触发时机：Game Over 时 / 打开暂停菜单时
PokiSDK.commercialBreak()
📺 播放广告。广告结束后自动继续游戏（返回 Promise）
📍 触发时机：Game Over 之后、下一局开始之前
📋 完整接入示例（让 AI 帮你改）
帮我在现有的 HTML 游戏里接入 Poki SDK： 1. 在 <head> 添加 SDK 脚本： <script src="https://game-cdn.poki.com/scripts/v2/poki-sdk.js"></script> 2. 游戏加载完成时调用： PokiSDK.gameLoadingFinished() 3. 玩家点击"开始"时调用： PokiSDK.gameplayStart() 4. 游戏结束时调用： PokiSDK.gameplayStop() 然后播放广告： await PokiSDK.commercialBreak() 广告结束后再允许重新开始 请在不破坏原有游戏逻辑的前提下完成接入。

📋 移动端适配提示词（让 AI 帮你加）
帮我给这个 HTML 游戏加上移动端支持： 1. 触屏跳跃：点击屏幕任意位置 = 跳跃（touchstart 事件） 2. 画面自适应：用 viewport meta 和 canvas 缩放， 确保在手机屏幕上完整显示不超出 3. 在屏幕下方加一个半透明的大"跳跃"按钮， 方便手机玩家操作（仅在触屏设备显示） 4. 防止手机滑动时误触发页面滚动 请确保电脑键盘操作依然正常。

进阶 05
性能优化：让游戏跑得更顺滑
▼
🎯 卡顿是玩家流失的第一原因
⚡ 性能优化清单（告诉 AI 帮你改）
🔄
用 requestAnimationFrame
不要用 setInterval 做游戏循环，用 requestAnimationFrame 让浏览器在最佳时机渲染
🎨
减少 Canvas 重绘
每帧只清除需要更新的区域，而不是 clearRect 整个画布
📦
对象池复用
障碍物不要每次 new 一个新对象，复用已经"死亡"的障碍物对象
🌐
Base64 内嵌图片
小图片转成 Base64 直接写进 HTML，省去网络请求时间
帮我优化这个 HTML 游戏的性能： 1. 把 setInterval 游戏循环改为 requestAnimationFrame 2. 用 delta time 让游戏速度与帧率无关 （速度 = 速度 × deltaTime / 16.67） 3. 障碍物用对象池复用，不要每次 new 4. 检查是否有内存泄漏（定时器没清理、事件监听没移除） 请保持游戏逻辑不变，只优化底层实现。
